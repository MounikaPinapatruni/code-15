DevOps Lifecycle

Plan

Define requirements, set goals and backlog items. Planning aligns stakeholders, identifies risks, and creates the roadmap for what to build next.

Develop

Write code, collaborate on design and perform code reviews. Developers implement features and fix bugs in short, iterative cycles

Build

Convert source into artifacts using CI: compile, package, and create deployable builds. Builds are reproducible and versioned.

Test

Run automated and manual tests (unit, integration, regression). Testing verifies quality and prevents regressions before release.

Release

Prepare and approve artifacts for production: tagging, release notes, and deployment gating. Releases coordinate the rollout plan

Deploy

Push release artifacts into production or staging environments. Deploys use automation (CD) to deliver changes safely and repeatably.

Operate

Run the application in production: scaling, backups, incident response, and ensuring reliability for end users.

Monitor

Collect metrics, logs and traces to measure performance and user experience. Monitoring triggers alerts and feeds the next planning cycle
